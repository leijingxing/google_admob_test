# 阅读器产品与商业化方案

## 1. 文档目的

本文档用于定义一个面向 Android / iOS 平台的漫画阅读器与小说阅读器工具项目方案，重点说明以下内容：

- 产品定位与目标用户
- 漫画阅读器功能设计
- 小说阅读器功能设计
- 书城、书架、详情页、阅读页的业务关系
- 广告变现与内购变现方案
- Android / iOS 双平台接入注意事项
- 适合当前 Flutter 模板项目的模块拆分建议

本文档面向产品、设计、客户端开发、测试和后续商业化接入人员，目标是让项目具备明确的功能边界和实现逻辑。

---

## 2. 项目定位

### 2.1 产品定位

本项目是一个跨平台阅读器工具项目，核心形态包括：

- 漫画阅读器
- 小说阅读器
- 书城与书架
- 广告与内购变现

项目本身不绑定某一个内容源，定位是“可快速演示、可继续业务化扩展”的 Flutter 阅读器基础架构。

### 2.2 目标

- 提供一套可演示的漫画阅读体验
- 提供一套可演示的小说阅读体验
- 兼容 Android 与 iOS
- 支持广告变现
- 支持内购去广告 / 会员 / 章节解锁
- 为后续接入真实内容接口、本地缓存、账号体系保留扩展点

### 2.3 目标用户

- 喜欢碎片化阅读漫画的用户
- 喜欢连续沉浸式阅读小说的用户
- 海外广告变现场景下的工具类 / 内容类 App 用户
- 需要较轻量内容消费和付费转化路径的用户

---

## 3. 产品结构总览

### 3.1 一级结构

建议保留 3 个底部 Tab：

- 书城
- 阅读 / 最近阅读
- 书架 / 我的

### 3.2 页面关系

整体流程建议如下：

1. 用户进入书城浏览作品
2. 点击作品进入详情页
3. 在详情页选择“开始阅读”
4. 根据作品类型进入漫画阅读页或小说阅读页
5. 阅读过程中触发广告、章节解锁、进度记录
6. 退出后回到书架或最近阅读

### 3.3 内容类型

建议将作品统一抽象为 `ReaderWork`，但在展示层区分两种阅读体验：

- `ComicWork`：以大图连续阅读为主
- `NovelWork`：以分页排版阅读为主

两者共享以下基础信息：

- 作品 ID
- 标题
- 作者
- 标签
- 简介
- 封面
- 章节列表
- 是否已收藏
- 最近阅读进度
- 是否已购买 / 是否会员可读

## 4. 漫画阅读器设计

### 4.1 核心目标

漫画阅读器要强调“低学习成本”和“沉浸式”，用户不需要复杂教学，进入页面后应自然滑动阅读。

### 4.2 页面组成

漫画详情页建议包含：

- 顶部悬浮信息栏
- 当前章节信息
- 漫画大图阅读区
- 右上角阅读设置入口
- 底部 Banner 广告位
- 章节结束后的下一章解锁卡片

### 4.3 交互原则

- 默认使用自然滑动翻页，不提供明显的“上一页 / 下一页”按钮
- 点击屏幕切换顶部 / 底部操作区显示与隐藏
- 支持图片缩放、拖拽查看细节
- 在章节结束时展示“下一章待解锁”提示
- 用户看完广告或完成购买后才能继续下一章节

### 4.4 阅读设置

漫画阅读设置建议放在右上角弹出面板，至少包含：

- 图片适配模式
- 适应宽度
- 铺满屏幕
- 阅读方向
- 纵向连续滚动
- 横向分页
- 预加载开关
- 自动隐藏顶部信息栏

### 4.5 漫画阅读细节

漫画页建议支持：

- 章节标题与副标题显示
- 阅读进度显示
- 当前页所属章节映射
- 最后一页触发解锁逻辑
- 本地缓存图片
- 弱网重试
- 图片加载失败占位

### 4.6 漫画详情页建议模块

- 封面区
- 标题 / 作者 / 标签 / 简介
- 收藏按钮
- 开始阅读 / 继续阅读按钮
- 章节列表
- 猜你喜欢 / 相关推荐
- 广告位

### 4.7 漫画商业化建议

漫画更适合以下变现路径：

- 底部横幅广告
- 章节结尾插页广告
- 激励广告解锁下一章节
- 会员免广告
- 单章或整本购买

原因是漫画天然有更明显的“章节切分点”，适合把广告放在阅读节奏节点，不破坏连续浏览体验。

## 5. 小说阅读器设计

### 5.1 核心目标

小说阅读器要强调“长时间停留”和“排版舒适度”，重点不是页面花哨，而是阅读稳定、设置完整、翻页顺滑。

### 5.2 页面组成

小说详情页 / 阅读页建议包含：

- 顶部标题栏
- 当前章节标题
- 文本阅读区域
- 右上角设置入口
- 底部广告位
- 番外 / 作者手记 / 附加内容解锁区域

### 5.3 阅读设置

小说阅读设置建议比漫画更完整，至少包含：

- 字体大小
- 行高
- 段间距
- 页边距
- 主题切换
- 纸张模式
- 夜间模式
- 护眼模式
- 翻页动画
- 仿真翻页
- 左右滑动
- 上下滚动
- 屏幕亮度跟随 / 独立调节

### 5.4 小说阅读细节

建议支持：

- 自动分页
- 按屏幕尺寸重新分页
- 保留阅读进度
- 章节切换
- 阅读历史
- 批注 / 书签
- 章节预加载
- 字体方案切换

### 5.5 小说详情页建议模块

- 封面区
- 标题 / 作者 / 类型 / 字数 / 更新状态
- 简介
- 目录
- 评论入口
- 加入书架
- 开始阅读 / 继续阅读
- 推荐作品

### 5.6 小说商业化建议

小说更适合以下变现路径：

- 阅读页底部横幅广告
- 章节切换时插页广告
- 激励广告解锁番外 / 附加内容
- 会员去广告
- 连载章节按章节付费
- 整本购买或订阅制

原因是小说阅读更连续，广告不能频繁打断正文，因此更适合低干扰横幅和节点式激励广告。

## 6. 书城、书架与阅读链路

### 6.1 书城

书城负责内容曝光和分发，建议包含：

- 推荐位
- 分类位
- 热门榜单
- 最新上架
- 漫画 / 小说双入口
- 搜索入口

### 6.2 书架

书架负责承接留存，建议包含：

- 已收藏作品
- 最近阅读
- 阅读进度
- 已下载章节
- 继续阅读快捷入口

### 6.3 阅读链路

建议统一为：

1. 书城进入详情
2. 详情决定阅读类型
3. 进入漫画或小说阅读页
4. 记录阅读进度
5. 章节末尾触发广告或购买
6. 返回书架继续管理

## 7. 广告变现设计

### 7.1 广告目标

广告的目标不是简单“多放广告”，而是：

- 提高用户生命周期价值
- 不明显破坏阅读体验
- 与章节节点、奖励节点结合
- 为后续内购转化提供对照路径

### 7.2 广告形式建议

阅读器项目建议使用以下广告形态：

- 开屏广告
- Banner 横幅广告
- 插页广告
- 激励广告
- 激励插页广告

原生广告可作为后续优化项，不建议一开始就放进阅读正文中，因为处理不好会严重破坏沉浸感。

### 7.3 广告位建议

#### 漫画阅读器

- 详情页底部：Banner
- 阅读页底部：Banner
- 章节结束：插页广告
- 下一章解锁：激励广告或插页广告

#### 小说阅读器

- 详情页底部：Banner
- 阅读页底部：Banner
- 章节切换：低频插页广告
- 番外 / 作者手记：激励广告

### 7.4 广告节流原则

必须控制广告频率，建议：

- 插页广告不连续触发
- 每次章节切换不一定都展示
- 激励广告必须由用户主动触发
- 新用户前几分钟减少强打断广告

### 7.5 广告回调处理

广告接入必须把回调做清晰，建议统一记录以下事件：

- `onAdLoaded`
- `onAdFailedToLoad`
- `onAdShowedFullScreenContent`
- `onAdDismissedFullScreenContent`
- `onAdFailedToShowFullScreenContent`
- `onAdImpression`
- `onAdClicked`
- `onUserEarnedReward`

建议处理方式：

- 加载成功：更新可展示状态
- 加载失败：记录日志并提供降级逻辑
- 展示失败：提示用户并重新请求广告
- 关闭广告：恢复阅读流程
- 激励完成：解锁章节、番外或奖励权益

### 7.6 广告状态机建议

建议每种广告位都有明确状态：

- 初始中
- 加载中
- 已加载
- 展示中
- 奖励完成
- 关闭完成
- 加载失败
- 展示失败

这样可以避免以下问题：

- 广告对象重复挂载
- 用户点击时广告未准备好
- 广告关闭后阅读状态错乱
- 跨页面复用同一个广告对象导致崩溃

### 7.7 Android / iOS 广告差异

广告接入时要明确双平台差异：

- Android 与 iOS 使用不同的 App ID
- 广告单元 ID 分平台管理
- iOS 需要额外处理隐私权限与审核说明
- Android 更容易先验证测试广告流程

测试阶段建议：

- Android 先完整跑通开屏、Banner、插页、激励
- iOS 再补权限说明、调试与审核文本

## 8. 内购变现设计

### 8.1 内购目标

内购不是替代广告，而是和广告配合：

- 广告负责广覆盖变现
- 内购负责高价值用户转化

### 8.2 内购形态建议

建议支持以下付费项目：

- 去广告月卡
- 去广告永久版
- 会员订阅
- 章节解锁包
- 单本购买
- 限时礼包

### 8.3 会员权益建议

会员可包含：

- 全站免广告
- 提前解锁章节
- 专属字体 / 主题
- 专属书架容量
- 云同步
- 离线缓存上限提升

### 8.4 广告与内购的关系

建议遵循以下逻辑：

- 非付费用户：可通过广告解锁部分内容
- 付费用户：直接享受免广告或会员内容
- 同一内容既可以广告解锁，也可以直接购买

这样可以降低用户第一次付费门槛，同时保留广告收入。

### 8.5 Android / iOS 内购差异

双平台接入要注意：

- Android 使用 Google Play Billing
- iOS 使用 Apple In-App Purchase
- 商品 ID 需要分平台统一规划
- 订阅状态需要服务端校验或本地安全校验策略
- 恢复购买在 iOS 是必备能力

### 8.6 购买恢复与容错

必须支持：

- 恢复购买
- 购买失败重试
- 取消购买处理
- 网络异常兜底
- 权益刷新

## 9. Android / iOS 平台注意事项

### 9.1 Android

- AdMob 广告流程调试相对直接
- 要区分测试广告与正式广告 ID
- 注意不同 Android 版本的网络与权限行为
- 打包时要关注混淆、签名、广告 SDK 版本兼容性

### 9.2 iOS

- 要配置 AdMob iOS App ID
- 要补充隐私用途说明
- 需要处理 ATT 授权弹窗策略
- 审核时广告和内购说明要清晰
- 恢复购买入口必须明显

### 9.3 隐私合规

无论 Android / iOS，都建议补齐：

- 隐私政策
- 用户协议
- 广告数据说明
- 个性化广告说明
- 内购退款说明

## 10. 工程实现建议

### 10.1 模块划分建议

结合当前项目结构，建议拆分为：

- `lib/modules/book_store/`
- `lib/modules/bookshelf/`
- `lib/modules/comic_reader/`
- `lib/modules/novel_reader/`
- `lib/modules/work_detail/`
- `lib/data/repository/reader/`
- `lib/data/models/reader/`
- `lib/core/ads/`
- `lib/core/iap/`

### 10.2 控制器职责建议

#### 漫画阅读控制器

负责：

- 图片分页与章节映射
- 广告加载状态
- 章节解锁
- 阅读进度
- 设置项状态

#### 小说阅读控制器

负责：

- 文本分页
- 字体、主题、行高设置
- 阅读进度
- 激励广告解锁番外
- 章节切换

#### 详情页控制器

负责：

- 作品信息展示
- 收藏状态
- 开始阅读 / 继续阅读
- 章节列表
- 相关推荐

### 10.3 广告封装建议

建议统一封装广告管理层，而不是把所有广告逻辑散在页面里。

可设计以下能力：

- Banner 广告管理器
- 插页广告管理器
- 激励广告管理器
- 广告状态日志
- 平台广告 ID 配置中心

### 10.4 内购封装建议

建议单独封装：

- 商品列表查询
- 发起购买
- 恢复购买
- 订阅状态判断
- 用户权益同步

## 12. 推荐业务规则

建议先确定以下业务规则，避免开发过程中反复推倒：

- 漫画下一章是否必须广告解锁
- 小说番外是否必须激励广告解锁
- 去广告是否同时去掉章节插页
- 会员是否包含章节免费
- 非会员是否允许广告解锁
- 书架与最近阅读是否需要云同步

这些规则一旦明确，广告、付费、阅读体验三者之间的逻辑才会稳定。

## 13. 当前项目落地方向建议

结合当前项目已经具备的基础能力，建议后续按下面顺序推进：

1. 先把漫画阅读器做成完整闭环
2. 再把小说阅读器的分页、主题、翻页动画打磨完整
3. 抽离统一广告管理层
4. 增加内购模块占位与商品模型
5. 再接真实内容源或本地内容包

这样做的好处是：

- 先把最直观的阅读体验跑通
- 再补商业化
- 最后再接复杂的数据来源

## 14. 结论

这个项目适合被定义为一个“跨平台阅读器工具底座”，而不是简单的广告 Demo。  
如果要让它具备继续业务化的价值，必须同时满足三件事：

- 阅读体验像产品，不像测试页
- 广告接入有边界，不乱放
- 内购和广告能共存，而不是互相冲突

对漫画阅读器，重点是沉浸式大图阅读、章节节点广告、下一章解锁逻辑。  
对小说阅读器，重点是排版、分页、主题、翻页动画和长期阅读舒适度。  
对商业化，重点是 Banner、插页、激励、去广告和会员体系的组合设计。
