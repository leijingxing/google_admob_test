这是一份详细的 `HttpService` API 请求使用文档，它将指导你如何在你项目的不同场景中使用这些网络请求方法。

---

## **HttpService API 请求指南**

### **一、 概述**

`HttpService` 是一个专为 Flutter 项目设计的、高度封装且易于使用的网络请求服务。它基于 `Dio` 库，并集成了以下核心特性：

*   **依赖注入**: 通过 `init` 方法配置，与业务逻辑完全解耦。
*   **自动解析**: 无需手动编写 `parser`，只需提供返回数据的泛型，即可自动将 JSON 转换为实体模型（Model）。
*   **统一结果处理**: 所有请求返回一个 `Result<T>` 对象，通过 `when` 方法可以优雅地处理成功与失败两种情况。
*   **内置功能**: 支持日志打印、请求缓存、文件上传和全局错误处理。

**核心原则：** 在发起请求时，你只需要关心 **“你要请求什么接口”** 和 **“你期望返回什么类型的数据”**。

### **二、 使用前提**

在发起任何请求之前，请确保你已完成以下两个步骤：

**1. 初始化 `HttpService`**

在你的应用入口文件（如 `main_dev.dart` / `main_prod.dart`）中，必须先调用 `init` 方法。

```dart
// lib/main_dev.dart 或 lib/main_prod.dart
Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final cacheDir = await getApplicationDocumentsDirectory();

  HttpService().init(
    baseUrl: 'https://your.api.server.com', // 替换为你的 API Base URL，末尾不要加 /
    cachePath: cacheDir.path,
    getToken: () async {
      // 实现获取用户 Token 的逻辑
      return "YOUR_USER_TOKEN";
    },
    onAuthError: (error) async {
      // 全局处理 401 认证失败，例如清理 Token 并跳转到登录页
      print("认证失败，跳转登录页！");
      await Get.offAllNamed('/login');
    },
  );

  runApp(const MyApp());
}
```

**2. 准备实体模型并注册**
代码生成器自动会自动生成有 JsonSerializable()注解的实体模型。
确保你的数据模型（Entity/Model）已创建，并且其 `fromJson` 方法已在 `json_parser.dart` 文件中注册。你的代码生成器应该会自动完成这一步。

```dart
import 'dart:convert';

import 'package:industrial_alarm/generated/json/base/json_field.dart';

import '../../generated/json/user_profile_entity.g.dart';

@JsonSerializable()
class UserProfileEntity {
  UserProfileEntity();

  factory UserProfileEntity.fromJson(Map<String, dynamic> json) =>
      $UserProfileEntityFromJson(json);

  ///	用户编号
  int id = 0;

  ///	用户账号
  String username = '';

  ///	用户昵称
  String nickname = '';

  ///	用户邮箱
  String email = '';


  Map<String, dynamic> toJson() => $UserProfileEntityToJson(this);

  @override
  String toString() {
    return jsonEncode(this);
  }
}

```

### **三、 两种推荐的请求方式**

目前项目内有两套互补的调用方式：

1. **直接使用 HttpService**：在控制器/服务层通过 `HttpService().get/post/...` 发起请求，自行管理路径、参数以及结果解析。这适合一次性脚本或复杂的流程控制。
2. **仓库层封装 + HttpAction 元数据**：推荐在 `lib/data/repository` 中为每个业务模块创建仓库类，混入 `HttpRepositoryMixin`，通过 `HttpAction` 描述接口元数据（路径、方法、缓存策略等），再由控制器调用仓库方法。这样仓库内只需声明“请求什么”，底层会自动拼装 URL、选择 HTTP 动词并复用统一的 HttpService。

#### HttpAction + HttpRepositoryMixin 快速入门

```dart
import 'package:industrial_alarm/core/http/http_action.dart';
import 'package:industrial_alarm/core/http/result.dart';

import '../models/alarm_change_apply_entity.dart';

/// 审批任务仓库
class ApprovalRepository with HttpRepositoryMixin {
  /// 默认构造函数。
  const ApprovalRepository();

  /// 审批任务列表接口
  static const HttpAction<List<AlarmChangeApplyEntity>> _approvalTasksAction =
      HttpAction<List<AlarmChangeApplyEntity>>(
    path: '/industrialAlarm/alarmChangeApply/applyPage',
    method: HttpMethod.post,
  );

  /// 获取审批任务列表
  Future<Result<List<AlarmChangeApplyEntity>>> getApprovalTasks({...}) {
    final data = {...}; // 请求体参数
    return requestAction<List<AlarmChangeApplyEntity>>(
      _approvalTasksAction,
      data: data,
    );
  }
}
```

`HttpAction` 支持以下常用字段：

| 字段 | 说明 |
| --- | --- |
| `path` | 接口路径，可包含占位符，例如 `/user/:id` |
| `method` | HTTP 动词，默认为 GET |
| `cachePolicy`/`cacheDuration` | GET 请求的缓存策略 |
| `pathBuilder` | 自定义路径拼接逻辑 |

在仓库中定义 `HttpAction` 常量可以避免每次请求都重新写字符串，也方便统一维护。`requestAction` 会根据 `method` 自动调用 `HttpService` 封装的具体实现，返回的仍然是 `Result<T>`，所以调用方处理逻辑不变。

**什么时候使用 HttpAction？**

- 同一个仓库中存在多处调用同一接口。
- 需要声明缓存策略或动态路径。
- 希望在仓库层只写“接口声明”，控制器只关心“仓库方法”。
- 常规业务接口优先使用 HttpAction，保证统一维护与一致行为。

**补充：直接使用 HttpService 依然适用的场景**

- 需要完全跳过 `Result<T>` 的解析，自己处理二进制/文件等特殊格式。
- 项目早期或脚本化需求，仓库封装成本较高。

### **四、 核心请求方法**

`HttpService` 提供了对 RESTful API 的完整支持。所有方法的返回值都是 `Future<Result<T>>`。

#### **1. GET 请求**

用于从服务器获取数据。

**方法签名:**
```dart
Future<Result<T>> get<T>(
  String path, {
  Map<String, dynamic>? queryParameters,
  Options? options,
  CancelToken? cancelToken,
  CachePolicy cachePolicy = CachePolicy.request,
})
```

**参数说明:**

| 参数 | 类型 | 描述 |
| :--- | :--- | :--- |
| `path` | `String` | **必需**。API 的路径，例如 `/users/123`。 |
| `queryParameters` | `Map<String, dynamic>?` | URL 查询参数，例如 `{'page': 1, 'limit': 10}`。 |
| `cachePolicy` | `CachePolicy` | 缓存策略，详见“高级功能”部分。 |
| `options` | `Options?` | Dio 的高级配置，通常不需要。 |
| `cancelToken` | `CancelToken?` | 用于取消请求，详见“高级功能”部分。 |

**使用示例:**

*   **获取单个对象:**
    ```dart
    Future<AlarmRespEntity?> fetchAlarmDetail(int id) async {
      final result = await HttpService().get<AlarmRespEntity>('/alarms/$id');
      
      return result.when(
        success: (alarm) => alarm, // 返回 AlarmRespEntity 对象
        failure: (error) {
          print("获取失败: ${error.message}");
          return null;
        },
      );
    }
    ```

*   **获取对象列表:**
    ```dart
    Future<List<AlarmRespEntity>> fetchAlarms({String? status}) async {
      final result = await HttpService().get<List<AlarmRespEntity>>(
        '/alarms',
        queryParameters: {'status': status}, // 示例：带上查询参数
      );

      return result.when(
        success: (alarms) => alarms, // 返回 List<AlarmRespEntity>
        failure: (error) => [], // 失败时返回空列表
      );
    }
    ```

#### **2. POST 请求**

用于向服务器提交数据以创建一个新资源。

**方法签名:**
```dart
Future<Result<T>> post<T>(
  String path, {
  dynamic data,
  Map<String, dynamic>? queryParameters,
  Options? options,
  CancelToken? cancelToken,
})
```
**参数说明:**

| 参数 | 类型 | 描述 |
| :--- | :--- | :--- |
| `path` | `String` | **必需**。API 路径。 |
| `data` | `dynamic` | **必需**。要发送的数据。通常是一个 `Map` 或一个调用了 `toJson()` 的实体对象。 |

**使用示例:**

*   **创建一个新的报警记录:**
    ```dart
    Future<bool> createAlarm(AlarmRespEntity newAlarm) async {
      // 假设接口创建成功后，返回创建的对象
      final result = await HttpService().post<AlarmRespEntity>(
        '/alarms',
        data: newAlarm.toJson(), // 发送实体对象的 JSON 格式
      );

      return result.when(
        success: (createdAlarm) {
          print("创建成功，ID 为: ${createdAlarm.id}");
          return true;
        },
        failure: (error) {
          print("创建失败: ${error.message}");
          return false;
        },
      );
    }
    ```

#### **3. PUT / PATCH 请求**

用于更新服务器上的一个现有资源。`PUT` 通常用于完整替换，`PATCH` 用于部分更新。

**方法签名 (以 PUT 为例):**
```dart
Future<Result<T>> put<T>(
  String path, {
  dynamic data,
  Map<String, dynamic>? queryParameters,
  Options? options,
  CancelToken? cancelToken,
})
```
**使用示例:**

*   **更新报警记录的标题:**
    ```dart
    Future<void> updateAlarmTitle(int id, String newTitle) async {
      // 假设接口返回更新后的对象
      final result = await HttpService().put<AlarmRespEntity>(
        '/alarms/$id',
        data: {'title': newTitle},
      );

      result.when(
        success: (updatedAlarm) => print("更新成功: ${updatedAlarm.title}"),
        failure: (error) => print("更新失败: ${error.message}"),
      );
    }
    ```

#### **4. DELETE 请求**

用于从服务器删除一个资源。

**方法签名:**
```dart
Future<Result<T>> delete<T>(
  String path, {
  dynamic data,
  Map<String, dynamic>? queryParameters,
  Options? options,
  CancelToken? cancelToken,
})
```
**使用示例:**

*   **删除一条报警记录:**
    ```dart
    Future<void> deleteAlarm(int id) async {
      // 假设接口删除成功后，返回一个简单的成功消息，例如: { "message": "Deleted successfully" }
      // 我们可以用 Map<String, dynamic> 或一个自定义的 SuccessResponse 模型来接收
      final result = await HttpService().delete<Map<String, dynamic>>('/alarms/$id');

      result.when(
        success: (response) => print("删除成功: ${response['message']}"),
        failure: (error) => print("删除失败: ${error.message}"),
      );
    }
    ```


### **五、 高级功能**

#### **1. 缓存策略 (`CachePolicy`)**

在 `GET` 请求中，你可以通过 `cachePolicy` 参数控制缓存行为。

*   `CachePolicy.request` (默认值): 仅发起网络请求，不读取也不写入缓存。
*   `CachePolicy.forceCache`: 优先从缓存读取。如果缓存存在，则直接返回缓存数据；如果不存在，则发起网络请求，并将成功的结果写入缓存。
*   `CachePolicy.refresh`: 强制发起网络请求，并将成功的结果写入缓存。这用于主动更新缓存。

**示例:**
```dart
// 获取新闻列表，优先使用缓存，减少不必要的网络请求
final result = await HttpService().get<List<NewsEntity>>(
  '/news',
  cachePolicy: CachePolicy.forceCache,
);
```

#### **3. 路径拼接注意**

- `baseUrl` 末尾不要带 `/`，`path` 以单个 `/` 开头。
- 避免出现 `//`，以免服务器路由解析异常。

#### **2. 取消请求 (`CancelToken`)**

当用户离开页面或执行一个新操作时，你可能需要取消正在进行的网络请求以避免不必要的资源消耗或状态更新。

**示例:**
```dart
class MyPageController extends GetxController {
  CancelToken? _cancelToken;

  void fetchData() {
    _cancelToken?.cancel(); // 取消上一次的请求
    _cancelToken = CancelToken();

    HttpService().get<MyData>(
      '/long_request',
      cancelToken: _cancelToken,
    );
  }

  @override
  void onClose() {
    _cancelToken?.cancel(); // 页面关闭时取消所有请求
    super.onClose();
  }
}
```
